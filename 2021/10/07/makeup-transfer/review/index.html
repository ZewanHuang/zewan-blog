妆容迁移综述

本文最后更新于：2021年10月11日 晚上

本篇主要介绍妆容迁移技术的概念和发展现状，以及在调研过程中提出的问题和发展方向。

1. 什么是妆容迁移

妆容迁移指的是，将一张人像的妆容(Reference)迁移到另一张人像照片中(Source)，合成新妆容的人像图片(Result)。

2. 传统妆容迁移算法

传统妆容迁移算法未使用深度学习技巧，往往对数据集具有较高的要求。根据要求的不同可以将传统妆容迁移算法分为两类：

  1. 需要成对的人脸妆造前后的对比图作为训练集，是有监督模型
  2. 不需要成对图，是无监督模型

2.1 基于梯度约束和成对数据的算法

基于成对图的算法，需要同一张人脸图像妆造前后的对比图作为训练集，对数据集的要求很高。此处以 Example-Based Cosmetic Transfer[1] 为例对该算法进行简要介绍。

如上图所示，$A$ 和 $A^{*}$ 是同一人像化妆前后的对比图数据，将 $A^{*}$ 的妆容迁移至一个全新的不同人脸图片 $B$ 上，获得 result 为 $B^{*}$。中心问题是，建立色彩和人脸表面几何的映射。

该妆容迁移算法 Cosmetic Transfer 由下列步骤组成：

  1. 预处理
  2. 妆容映射
  3. 外观修正
  4. 眼部迁移

首先是人脸预处理步骤，通常需要

  • 去除眉毛和眼睫毛
  • 使用纹理合成填充空缺处
  • 提取内在皮肤特征，如雀斑等 target face 需要保留的特征和 reference face 需要去除的特征
  • 将脸部几何扭曲转化为经典的人脸模型

具体如何实现上述操作，在此处不再赘述，论文使用的是前人的成果

第二步是妆容映射，主要是建立 $A$ 和 $A^{*}$ 化妆前后的 color 和 reflectance 的映射关系 $C$，并将其作用到 $B$ 上。

映射关系建立较为简单，对人脸图片按各个像素点进行色彩特征量化，并将对应的像素点做如下公式运算，解得每个像素点的映射关系：

$$
c_{p}=a_{p}^{*}/a_{p}
$$

获得各像素点映射关系，组成总映射 $C$，将其应用到 $B$ 上：

$$
b_{p}^{*}=c_{p}b_{p}
$$

需要注意的是，完成 $B$ 的初步映射后，需要将第一步提取的内在皮肤特征贴合到 $B$ 上。

当然，$A$ 与 $A^{*}$、$A^{*}$ 与 $B$ 脸部的几何偏差，会对第二步建立的像素级映射带来较大的误差，此时需要第三步，外观矫正。

该步骤主要是在得到的映射关系上，对 $B$ 在像素级别上的色彩强度进行优化，优化公式主要有：

$$
b_{p}^{*}=b_{p}(\gamma(c_{p}-1)+1)\tag{1}
$$

$$
\Delta(b^{*}_{p})=\Delta(\beta b_{p}+(1-\beta)a_{p}^{*})\tag{2}
$$

公式(1)引入 $\gamma$，当值为 0 时表示无妆容，当值为 1 时表示妆容完全迁移，因此 $\gamma$ 控制迁移程度。

公式(2)引入拉普拉斯算子，使用 Gauss-Seidel 求解器求解 reference face 和 source face 之间的权重，以优化参考人脸和目标人脸的几何关系。

最终在公式(1)和(2)的约束下求解优化，并将眉毛、眼睫毛等眼部特征迁移至人脸中，得到 $B^{*}$。

该方法有较大的局限性：

  1. 要求 source 和 reference 肤色相近，背景单一；
  2. 无法适应较大的几何偏差，即 source 和 reference 人脸位置偏差大时迁移效果差；
  3. 数据集为成对的妆容对比图，要求高。
